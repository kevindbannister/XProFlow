## Screenshot workflow for Codex agents
When taking screenshots for UI changes, do **not** capture the login screen unless the change is specifically about authentication.

Before capturing screenshots, always:
1. Open the app.
2. Log in with:
   - **username/email:** `master`
   - **password:** `master`
3. Navigate to the page/feature that was changed.
4. Capture the screenshot(s) from that in-app location.

If login fails, report the failure in the final message and include what blocked successful authentication.
